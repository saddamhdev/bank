## Short Answer

❌ No — Maven does NOT create 3 physical JAR copies in your project.

But ✅ each service will have its own copy in its runtime classpath.

Let’s break it clearly.

---

# 🏗 Scenario

```
parent
 ├── common-lib
 ├── service-a
 ├── service-b
 └── service-c
```

All three services depend on `common-lib`.

---

# 🔎 What Happens During Build?

### Step 1: Maven builds `common-lib`

It generates:

```
common-lib-1.0.0.jar
```

This JAR is stored in:

```
~/.m2/repository/com/example/common-lib/1.0.0/
```

👉 Only ONE jar file exists in local Maven repository.

---

# 🔎 What About service-a, b, c?

Each service:

* Does NOT create new jar of `common-lib`
* Just references the same jar from `.m2` repository

So physically:
✔ One jar in `.m2`
✔ Not three separate physical builds

---

# 🚀 But At Runtime?

When you package each service:

```
mvn clean package
```

Each Spring Boot service creates its own fat jar:

```
service-a.jar
service-b.jar
service-c.jar
```

Inside each fat jar, `common-lib` classes are included.

So:

| Level            | Copies                  |
| ---------------- | ----------------------- |
| Maven local repo | 1                       |
| Running services | 3 (one inside each app) |

This is normal and required because:

Each microservice runs independently.

---

# 🧠 Think Like This

Development time:

> One shared library

Runtime:

> Each service becomes self-contained application

---

# 🔥 Important Difference

If it is true distributed microservices:

Each service MUST have its own copy at runtime.

If it is monolithic multi-module app:

Only one final application jar will be created.
