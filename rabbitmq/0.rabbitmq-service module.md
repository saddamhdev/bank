Since you already have a **rabbitmq-service module**, the architecture should be clean like this:

```
gateway-service  →  rabbitmq-service  →  splunk-service  →  Splunk
```

NOT:

```
gateway → direct queue → splunk
```

Let’s design it properly.

---

# 🎯 Correct Microservice Responsibility

### ✅ gateway-service

Only publishes log event (producer)

### ✅ rabbitmq-service

Central messaging layer (exchange + queue management)

### ✅ splunk-service

Consumes log queue and sends to Splunk

This keeps everything modular and shareable.

---

# 🔥 STEP 1 — gateway-service (Producer Only)

Inside your `ApplicationDetailsGlobalFilter`, instead of sending to Splunk, send to RabbitMQ.

Inject publisher:

```java
private final RabbitTemplate rabbitTemplate;
```

Constructor:

```java
public ApplicationDetailsGlobalFilter(ObjectMapper objectMapper,
                                      Tracer tracer,
                                      RabbitTemplate rabbitTemplate) {
    this.objectMapper = objectMapper;
    this.tracer = tracer;
    this.rabbitTemplate = rabbitTemplate;
}
```

Inside `.doFinally()`:

```java
Map<String, Object> logEvent = new HashMap<>();
logEvent.put("service", "gateway-service");
logEvent.put("path", mutatedRequest.getURI().getPath());
logEvent.put("method", mutatedRequest.getMethod().name());
logEvent.put("status", exchange.getResponse().getStatusCode());
logEvent.put("durationMs", duration);
logEvent.put("correlationId", correlationId);
logEvent.put("traceId", resolvedTraceId);
logEvent.put("spanId", resolvedSpanId);
logEvent.put("timestamp", Instant.now());

rabbitTemplate.convertAndSend(
        "central.exchange",
        "log.routing",
        logEvent
);
```

Done.

# 🔥 STEP 2 — rabbitmq-service (Messaging Hub)

Inside rabbitmq-service:

### 1️⃣ Define exchange + queue

```java
@Configuration
public class MessagingConfig {

    @Bean
    public TopicExchange centralExchange() {
        return new TopicExchange("central.exchange");
    }

    @Bean
    public Queue logQueue() {
        return new Queue("log.queue");
    }

    @Bean
    public Binding binding(Queue logQueue, TopicExchange centralExchange) {
        return BindingBuilder.bind(logQueue)
                .to(centralExchange)
                .with("log.routing");
    }
}
```

That service manages messaging infrastructure.

---

# 🔥 STEP 3 — splunk-service (Consumer)

Inside splunk-service:

```java
@RabbitListener(queues = "log.queue")
public void consumeLog(Map<String, Object> logEvent) {
    splunkHecService.sendEvent(logEvent).subscribe();
}
```

Done.

# 🎯 Why This Is Correct Architecture

Because now:

| Service          | Responsibility        |
| ---------------- | --------------------- |
| gateway          | produce logs          |
| rabbitmq-service | message broker config |
| splunk-service   | log processor         |

Everything is cleanly separated.
