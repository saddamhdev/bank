# 🌍 What Does “External Client” Mean?

An **external client** is:

> Any system **outside your backend microservice cluster** that sends requests to your system.

It is not part of your internal services.

---

# 🔹 Simple Architecture Example

```text
Mobile App / Browser  →  API Gateway  →  Internal Services
   (External Client)         (Your System)
```

The Mobile App and Browser are **external clients**.

---

# 🔹 Examples of External Clients

### 1️⃣ Web Browser

* Chrome
* Edge
* Firefox

User opens:

```
https://yourbank.com
```

Browser = External Client

---

### 2️⃣ Mobile App

* Android banking app
* iOS e-commerce app

The app calls:

```
https://api.yourbank.com/login
```

App = External Client

---

### 3️⃣ Third-Party Systems

* Payment gateway
* Another company’s API
* Partner system

If they call your public API → they are external clients.

---

# 🔹 Internal vs External

| Type             | Example       | Inside Your Microservices? |
| ---------------- | ------------- | -------------------------- |
| External Client  | Mobile App    | ❌ No                       |
| External Client  | Browser       | ❌ No                       |
| Internal Service | user-service  | ✅ Yes                      |
| Internal Service | order-service | ✅ Yes                      |

---

# 🔹 Why This Matters (REST vs gRPC)

External clients usually use:

👉 REST (JSON)
Because:

* Easy to debug
* Works in browser
* Human readable
* Standard over internet

Internal services (inside your system) may use:

👉 Feign (REST)
👉 gRPC (high performance)

---

# 🔥 Real Banking Example

Imagine:

```text
Customer App → Gateway → Account Service → Fraud Engine
```

Customer App = External Client
Fraud Engine = Internal Service

You expose REST to external world.
You may use gRPC internally.

---

# 🎯 Final Clear Definition

External Client =
Any application **outside your backend system boundary** that consumes your API.

Internal Client =
A service inside your architecture calling another service.
